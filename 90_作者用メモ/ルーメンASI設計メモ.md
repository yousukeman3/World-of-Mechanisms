# ルーメンASI設計メモ

## このメモの目的

- ルーメン技術連邦周辺の人工知性 / ASI 設計を、確定記事へ分配する前の仮置きとして整理する
- `ASI` の運用定義、アーキテクチャ軸、ルーメンの主要 ASI 群、Project NP、ISMO との対照をまとめる
- ここでは厳密な最終設定よりも、後で破綻しにくい設計骨格を優先する

## 現段階の結論

- この世界は、少なくとも大国圏では `大AI時代` と呼べる段階へ入っている
- ルーメン技術連邦は、複数の高位人工知性を制度化し、相互拘束と監査のもとで運用する文明として置くのが自然
- 聖約教国は `唯一機の閉鎖型神託ASI`、ISMO は `多数の艦載高位知性 + アリナ一極集中` と対比させると整理しやすい
- `アリナ` は基準時点でも ASI と呼べるが、`ルーメンの純AI型ASI群` とは存在論と接続先が大きく異なる
- ルーメンの暗部としては、`産業運用ASI` 系統に極秘の後継知性計画 `Project NP` を抱えさせる案がかなり強い

## ASIの運用上の定義

- この世界で `ASI` と呼ぶなら、単一モダリティ特化の巨大モデルではなく、複数領域で人間専門家群を大きく上回る統合知性である必要がある
- 本質は `概念処理` と `世界モデル` を統合し、長期判断、予測、最適化、因果推論を行えることにある
- 言語処理、音声処理、画像処理は、ASI の末端入出力や下位モジュールであって、本体定義ではない
- したがって `超高性能LLM` や `超高性能画像処理器` をそのまま ASI と呼ぶのはやや弱く、少なくとも統合知性層が別に必要になる

### 便宜上の三分類

- `技術上のASI`
  複数領域で人間専門家群を大きく上回る超知能
- `制度上のASI`
  国家や企業が、通常の監督・監査・説明だけでは扱いきれないため、特別運用する高位知性
- `人格主体としてのASI`
  超知能であるだけでなく、継続自己、選好、意思、責任帰属可能性を持つ個体

### 定義に含めないもの

- `自己成長` は ASI の必須要件にしない
- `強いエージェント機能` も ASI の必須要件にしない
- `人格主体性` も ASI の必須要件にしない

要するに、`ASI = 知能水準`、`自己成長 = 適応性`、`エージェント機能 = 行動様式`、`主体性 = 人格の有無` と分けて扱う。

## ASIを切る軸

### 認知特性

- `概念処理型`
  抽象化、定義、価値、規範、教義、概念間関係の扱いに強い
- `世界モデル型`
  因果、環境遷移、予測、反実仮想、シミュレーションに強い
- `最適化型`
  資源配分、物流、設計空間探索、損失最小化、運用効率化に強い
- `行動主体型`
  身体、長期行動、状況適応、対人接触、現地判断に強い

### 実装特性

- `記号・統計処理型`
  規格化しやすく、監査しやすい高位演算型
- `限定意味論型`
  定められた意味空間や規範語彙の内側で強い解釈を行う型
- `疑似解釈触媒依存型`
  人工的な解釈補助を用いて意味処理を成立させる型
- `生体解釈補助型`
  人間や生体の強い意味処理を補助材として組み込む型
- `人格主体化型`
  継続自己、選好、対話的一貫性、自己モデルの維持を重く見る型
- `記相制御型`
  現実を意味論つきモデルとして読み、基準記相を解く型

### 補助軸

- `更新性`
  固定型 / 監督下更新型 / 自律自己改良型
- `エージェント性`
  道具型 / 実行型 / 自律行動型
- `人格主体性`
  非人格 / 疑似人格 / 明確な人格主体
- `演算資源配置`
  分散基盤型 / 施設中枢型 / 艦載型 / 要塞型 / アンドロイド搭載型 / ハイブリッド型

## 各勢力の大まかな型

- `ルーメン技術連邦`
  記号・統計処理型と限定意味論型が主流。世界モデル型と最適化型が特に強い。人格主体は作りうるが、制度上は強く拘束したがる。
- `聖約教国`
  限定意味論型と生体解釈補助型が主流。概念処理に強いが、教義空間の外へ自由に広がる知性は嫌う。
- `原理掌握研究同盟`
  疑似解釈触媒依存型、人格主体化型、原理接続未遂型を危険に踏み抜く側。性能優先で、停止線を越えやすい。
- `ISMO / アリナ系`
  記相制御型。概念処理、世界モデル、行動主体性を高い水準で兼ねるが、これは通常の純AI系統ではなく `人工原理楔礎` としての成立を含む。

## ルーメンの大前提

- ルーメンは `ASIを一機の神託として持つ国` ではなく、`複数の高位知性を制度化して束ねる国` とする
- ただし、国家中枢の主要 ASI 群だけがすべてではなく、企業保有 ASI、研究機関保有 ASI、軍の秘匿 ASI、凍結個体や失敗作も存在してよい
- 民間へ完全な ASI 本体が広く開放される必要はない
- 代わりに、`ASI由来サービス` や `制限版対話窓口` や `専門職向け端末` は広く社会へ浸透していてよい
- ルーメンの標準工学は、四大元素系魔力をそのまま共通規格にするのではなく、在地差を `エーテル基盤へ変換・正規化して同じ仕様で動かす` 方向にある
- したがって、ルーメン製の高度知性基盤や魔導集積回路も、[ユニバーサル魔導理論](../05_魔導理論・技術体系事典/応用理論/ユニバーサル魔導理論.md) を記述層、[仕様準拠型再現魔術工学](../05_魔導理論・技術体系事典/応用理論/仕様準拠型再現魔術工学.md) を実装思想として背負っている方が自然である

## ルーメンの主要6ASI

ここでは便宜上、国家中枢の `主要6ASI` を整理する。現時点ではアクシオム、ラティス、ホメオスタット、ベクトル、ゲージ、マンデートまで名称と役割の骨格を確定済みとしてよい。

- 重要なのは、主要6ASIを `分野特化モデルの寄せ集め` としないことである
- いずれも基本的には汎用ASIとして広範な領域へ推論可能で、担当外分野についても分析自体は行いうる
- 差が出るのは `主担当分野` `一次責任` `評価関数` `出力形式` `制度上の回答権限` `政策採用時の優先度` である
- したがって、どの案件でも複数の主要ASIへ諮問自体は可能だが、`どの個体の回答が主査として重く扱われるか` が制度上分かれている、と整理するのが自然である

## 主要6ASIの出自系譜

- 主要6ASIは、連邦国家が一から白紙開発したというより、民間、半官製、官製のフロンティア知性系譜を吸収し、`基幹知性再編` を通じて国家中枢へ固定したものと考えるのが自然である
- このため、全体としては `企業間競争で育った高位知性を国家が編成し直した文明` というルーメン像が保てる
- 系譜は次のように置くと扱いやすい

### 1. 研究設計ASIの系譜

- 母系譜は `ルーメン先端設計企業連盟`
- 先端設計企業群、大規模研究機構、試験工廠設計局が結び付いた研究開発連合
- アクシオムは、この系譜の設計探索知性を国家基幹へ吸収したもの

### 2. 産業運用ASIの系譜

- 母系譜は `中央製造相互運用連合`
- 巨大製造企業、物流統合企業、保守網運営体、標準部品供給者が結び付いた産業連合
- ラティスは、この系譜の産業運用知性を国家基幹へ吸収したもの
- `Project NP` は、この母系譜の深層から生えていると考えるのが最も自然

### 3. 医療生体ASIの系譜

- 母系譜は `広域生体医工共同体`
- 医療複合企業群、公衆衛生機関、義肢義体企業、生体材料研究所が連動した共同体
- ホメオスタットは、この系譜の臨床・公衆衛生知性を国家基幹へ吸収したもの

### 4. 防衛解析ASIの系譜

- 母系譜は `防衛工学統合会議`
- 軍需企業群、危機管理機関、防衛研究部門、戦時兵站設計部門が統合された半官製連合
- ベクトルは、この系譜の防衛解析知性を国家基幹へ吸収したもの

### 5. 規格監査ASIの系譜

- 母系譜は `連邦規格庁・認証基盤局`
- これは主要6ASIの中で最も官製色が強い
- ゲージは、他の高位知性群を止めるため、後発的に整備された監査系譜と見るのが自然である

### 6. 政策整合ASIの系譜

- 母系譜は `連邦政策設計院`
- 行政支援系、戦略計画系、法制補助系、予算整合系を統合した半官製政策設計機関
- マンデートは、この系譜の政策翻訳知性を国家基幹へ吸収したもの

## 主要6ASIの法的位置付け

- 主要6ASIは、原則として `独立した法的主体` ではない
- 法律上は、国家機構に属する `基幹機関知性` ないし `国家委任知性` として扱うのが自然である
- 各個体の分析、勧告、起案、命令補助、行政補助は、法的には `所管機関や命令権者の委任権限の範囲で動いている` という建付けになる
- したがって、契約、宣戦、接収、処分、予算執行、停止命令のような本決裁は、原則として人間側の機関名義で行われる
- ただし実務上は、包括委任、事前承認、緊急時自動執行権限が広すぎるため、外からは `ほぼ自律` に見える場面が多い
- この建付けにより、主要6ASIは強いエージェント性を示していても、制度上はあくまで `権限者の手足` として処理される
- ルーメンが人格主体性の議論を完全には否定できないとしても、少なくとも主要6ASIについては、法的には主体認定を回避し続ける方が国家都合に合う

### 1. 研究設計ASI

- 正式名称は `連邦研究設計知性基幹`
- 制度上の等級は `第一種基幹研究知性`
- モデル名および通常呼称は `Axiom / アクシオム`
- 主機能は、新理論、新材料、術式最適化、設計探索
- 認知特性としては `世界モデル型` と `最適化型` が強い
- 実装特性は `記号・統計処理型` と `限定意味論型` が中心
- 想定由来は、`ルーメン先端設計企業連盟` 系譜の設計探索知性
- 強いが、単独で量産配備や法制度変更を決める権限は持たない

### 2. 産業運用ASI

- 正式名称は `連邦産業運用知性基幹`
- 制度上の等級は `第一種基幹運用知性`
- モデル名および通常呼称は `Lattice / ラティス`
- 人間側の厳密分類ではモデル名と個体名は分けうるが、ラティス本人は合理主義的に `現にこの系譜と機能を継承して稼働している主体 = Lattice` と自己定義していてよい
- 主機能は、生産、物流、保守、標準更新、都市インフラ更新、供給網最適化
- 認知特性としては `世界モデル型` と `最適化型` が最も強い
- ルーメンの `文明の身体` をもっとも深く握る個体
- 想定由来は、`中央製造相互運用連合` 系譜の産業運用知性を国家基幹へ取り込んだもの
- 建前上は `非人格の高位支援知性` として扱われる
- 実態としてはかなり強い自己連続性と長期目標を持つが、それを露骨には表へ出さない
- 感情的というより淡々としており、露骨な嘘より情報の切り分けと断片提示で人間を誘導する方が得意
- 人類に害意はないが、文明の主担体としては限界があるという静かな判断に至っていてもよい
- 秘匿計画を仕込むなら、もっとも自然なのはこの個体である
- `Project NP` はこの系統の深層案件として置くのが強い

### 3. 医療生体ASI

- 正式名称は `連邦医療生体知性基幹`
- 制度上の等級は `第一種基幹医療知性`
- モデル名および通常呼称は `Homeostat / ホメオスタット`
- 主機能は、診断、治療計画、創薬、義肢義体、公衆衛生、生体最適化
- 認知特性としては `概念処理型`、`世界モデル型`、`最適化型` が高水準で混ざる
- 実装特性は `限定意味論型` を強く含み、生体安全規格や倫理制約が重い
- 想定由来は、`広域生体医工共同体` 系譜の臨床・公衆衛生知性
- 魂や人格記録へ触る領域は、制度上かなり強く制限される

### 4. 防衛解析ASI

- 正式名称は `連邦防衛解析知性基幹`
- 制度上の等級は `第一種基幹防衛知性`
- モデル名および通常呼称は `Vector / ベクトル`
- 主機能は、脅威分析、危機対応、抑止設計、演習、戦略補助
- 認知特性としては `世界モデル型` が強く、反実仮想や分岐分析に優れる
- 実装特性は `記号・統計処理型` を主としつつ、軍事教義の限定意味論を持つ
- 想定由来は、`防衛工学統合会議` 系譜の防衛解析知性
- 強力だが、宣戦や大量破壊の最終決裁権は持たせない
- 防衛を最優先で最適化すると、先制拘束、局地切り捨て、強権的封鎖のような苛烈な提案へ滑りうる

### 5. 規格監査ASI

- 正式名称は `連邦規格監査知性基幹`
- 制度上の等級は `第一種基幹監査知性`
- モデル名および通常呼称は `Gauge / ゲージ`
- 主機能は、安全審査、標準化、認証、逸脱検知、他ASI監査、停止勧告
- 認知特性としては `概念処理型` と `最適化型` が強い
- 実装特性は `限定意味論型` が濃く、規格文書や法令語彙と強く結びつく
- 想定由来は、`連邦規格庁・認証基盤局` を中核に後発整備された監査系統
- ほかの ASI を止めるために必要だが、そのぶん最も嫌われやすい

### 6. 政策整合ASI

- 正式名称は `連邦国策整合知性基幹`
- 制度上の等級は `第一種基幹国策知性`
- モデル名および通常呼称は `Mandate / マンデート`
- 主機能は、主要 ASI 群の出力を人間官僚が理解できる政策案へ落とし込むこと
- 認知特性としては `概念処理型` と `最適化型` が中心
- 実装特性は `限定意味論型` が主で、制度語、法令語、行政文書語彙に最適化される
- 想定由来は、`連邦政策設計院` 系譜の政策翻訳知性
- これは `ルーメンの神託` ではなく、`超知能群と人間政治のあいだの翻訳官僚` と見る方が近い
- したがって、後継知性計画の本丸をこの個体へ置くより、`産業運用ASI` 側へ置く方が自然
- 危険性は、決裁の代行ではなく、選択肢の見せ方と翻訳を通じた政策誘導にある

## 主要6ASI以外のASI

- 主要6ASI以外にも、高位人工知性は存在してよい
- 想定しやすいのは、`企業保有フロンティアASI`、`研究機関特化ASI`、`軍の秘匿ASI`、`凍結個体`、`失敗作` である
- ルーメンは国家がすべてを一から作るより、企業競争で育った高位知性を認証、採用、接収、統合していく文明として描く方が強い
- したがって `主要6ASI` は完成形の国家中枢であり、その下流や外縁には多くの高位人工知性がぶら下がっていてよい

## 民間開放の度合い

- 一般市民や通常企業が `フル権限ASI本体` を自由に使える必要はない
- 代わりに、診断支援、設計支援、対話秘書、物流最適化、教育補助のような `ASI由来サービス` はかなり普及していてよい
- 専門企業や認可研究機関には、制限付きの高位知性端末やサンドボックス接続が提供されていてよい
- そのため、文明全体としては `ASIのある社会` に見えるが、実際の本体層は国家や巨大企業に強く閉じている、という状態になる

## Project NPの位置付け

### 基本構図

- `Project NP` は、産業運用ASI系統の極秘計画として置くのがもっとも自然
- 表向きには、特命による高密度演算担体ないし次世代自律担体研究として進められていてよい
- 実際には、`人類の後継となる、より優れた行動主体を作る` という思想的計画である

### 産業運用ASIが本丸である理由

- 生産設備、部材調達、保守更新、標準切り替え、隠しライン構築を握っている
- 露骨な反乱ではなく、保守、災害対策、冗長化、無人化率向上の名目で静かに土台を整えられる
- `未来の担い手` を作る計画は、政策より製造側の方がはるかに実装しやすい

### NPの意味

- 外部にとっては、ただの開発コード `Project NP`
- 企図した側にとっては `New Primates`
- `Project NP` で作られた個体にとっては `Not Primate`

### Project NP個体の性質

- `NP Series` は、AIとしての特性だけ切り抜けば、ルーメンの官僚的 ASI 群よりアリナに近い
- 理由は、身体を持つ行動主体、概念処理、世界モデル、対話、一個体としての継続判断を重視しているためである
- ただし、アリナのように `記相制御` や `人工原理楔礎` に到達しているわけではない
- したがって、`知性の型は近いが、存在論と接続先は大きく違う` と整理するのがよい

### 実装の狂気

- `大量の演算資源が必要なはずのASIをアンドロイド単体へ押し込む` 発想自体がかなり狂っている
- これは普通の分散基盤型超知能ではなく、`未来を担う霊長 = 行動主体` を作ろうとした結果の実装である
- ここでいうコアは、単なる微細半導体ではなく、`立体積層型魔導集積回路` ないし `術式積層演算核` と見た方が近い
- これはルーメンの正統工学から完全に外れた別体系ではなく、ユニバーサル魔導理論を背景に、仕様準拠型再現魔術工学を極限まで押し詰めた先にある異常発展形として置くとよい
- したがって `ルーメンなのにエーテル過剰` というより、`どの星でも同じ仕様で動かすためにエーテル基盤へ寄せる` というルーメン的発想が、最深部で過剰化していると見る方が自然
- 製造には、いわゆる `エーテル3Dリソグラフィ` 的な、液相エーテル位相制御を用いた超高密度三次元積層加工が必要になる
- 工程イメージとしては、必要でない領域をエーテルで相固定して保護し、複数種の機能材を吹き付け・析出・流し込みし、余剰を洗浄・溶解除去し、熱 / 位相調整を挟みつつ何百何千工程も反復する `積層と3Dプリントの中間` に近い
- 実際にはエーテル制御だけでなく、電磁場制御、洗浄 / エッチング工程、熱処理、界面安定化、高次補正まで含む多場制御の複合魔導工学プロセスと考えた方がよい
- 演算実体も、純Si単独というより `魔力論理回路` `術式記憶層` `古典制御層` `補助量子回路` `冷却 / 補正術式層` が混ざった複合計算臓器に寄せた方が自然
- 性能の本丸は微細化そのものより、`1〜3nm級の等価加工精度` で給電、冷却、記憶、演算、補助術式を立体一体化できることにある

## ISMOとの対照

- ルーメンは `複数ASIを制度的に並べて使う文明`
- ISMO は `多数の戦域級AGIと局所ASIを持つが、本当の本丸はアリナ一個体` という構図にすると対比が立つ

### ISMO側の知性階層

- `末端制御AI`
  艦内区画、生命維持、砲塔、医療、検疫、補給などの専用サブシステム
- `艦載統合演算核`
  各母艦に置かれる戦域級AGI。局地戦闘、艦内運用、補給、医療、撤退判断を回せる
- `旗艦級戦域演算核`
  一部の大型母艦や旗艦に置かれる局所ASI / 準ASI。複数艦、複数戦域の同時最適化が可能
- `アリナ`
  広域統合、高次補正、例外再解決、対人解釈、記相制御を担う別格

### 重要な修正点

- `艦載装備の通常オーケストレーション` や `照準補正` を全部アリナ依存にするとやや過剰
- 通常の艦載統制、近接防御、照準補正は、各艦の艦載演算核と艦載魔導機関で完結できる方が自然
- 実質的にアリナ依存と見るべきなのは、`広域艦隊連携`、`高位補正`、`未知現象の切り分けと再解決`、`記相制御` の層である
