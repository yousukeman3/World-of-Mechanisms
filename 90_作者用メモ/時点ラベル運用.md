# 時点ラベル運用

記事の `いつ時点の情報か` を揃えるための運用メモ。
設定が変わる対象は、本文とは別に `基準時点` と `変遷` を管理する。

## 基本方針

- `恒常設定` と `時点依存の状態` を混ぜない
- 冒頭の `管理メタ` で必ず `基準時点` を示す
- 変化が大きい対象は `変遷` か別状態記事で管理する
- `現在` という表現だけで済ませず、どのラベル時点の現在かを示す

## 推奨メタ項目

```md
## 管理メタ

- 基準時点:
- 公知度:
- 主な認知層:
- 真偽:
- 変動性:
- 時点依存:
- 例外的認知者:
```

## 基準時点の候補

- `神話時代`
- `古代`
- `前近代`
- `近代`
- `現代`
- `物語開始前`
- `物語開始時`
- `第一部後`
- `終盤`

必要に応じて、

- `ISMO成立前`
- `ISMO成立後`
- `星間進出前`
- `星間進出後`

のような補助ラベルを併用してよい。

## 変動性ラベル

### 低

- 理論や根本設定のように、時代によって変わりにくい

### 中

- 人物の肩書き、組織の構成、社会通説のように、変わるが頻繁ではない

### 高

- 戦況、勢力図、公知情報、現役の役職、条約状態など

## 時点依存ラベル

### 低

- ほぼ恒常設定で読める

### 中

- 基準時点を見ないと誤読しやすい

### 高

- 状態ページや変遷節がないと破綻しやすい

## 本文構成の推奨

状態変化のある記事では、以下の順が扱いやすい。

```md
## 恒常設定
## 基準時点の状態
## 変遷
## 関連事件
```

## 別記事に分ける目安

以下のような対象は、時点ごとの状態を別記事にしてよい。

- 国家
- 軍事組織
- 大規模勢力
- 現役の政治指導者
- 戦線
- 公知/機密の境界が大きく変わる情報

例:

- `星間治安維持機構（ISMO）.md`
- `星間治安維持機構（ISMO）_物語開始時.md`

## 運用上の注意

- `恒常設定` に現在の地位や戦況を書かない
- `物語開始時` のようなラベルは、後から章単位ラベルへ置換してよい
- 年号が未確定でも、相対時点ラベルだけで十分運用できる
