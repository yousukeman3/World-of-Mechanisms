# 術式パッケージと配備束

## 管理メタ

- 基準時点: 現代
- 公知度: 専門家公知
- 主な認知層: 再現魔術士 / 魔導技術者 / 艦載整備士 / 医療術式管理官 / ISMO技術研究開発局
- 真偽: 通説中心
- 変動性: 中
- 時点依存: 中
- 例外的認知者: アリナ

## 基本情報

- 名称: 術式パッケージと配備束
- 分類: 基盤技術 / 配備設計 / 術式実装単位
- 開発勢力: 複数文明圏 / ISMO系技術圏 / 港湾・工廠系技術者集団
- 主用途: 標準術式の再利用、装備組込、艦載常駐制御、認証配備
- 普及度: 中
- 危険度: 高

## 概要

術式パッケージとは、特定任務に必要な術式群、基準記相参照、安全制約、触媒調律、接続規格をまとめた `論理的な機能束` である。これに対し配備束とは、その術式パッケージを `どの術者、どの装備、どの艦載基盤へ、どの認証条件で実装するか` まで確定させた配備単位である。

この区別によって、同じ機能中核を人間用と魔導機械用へ分けて載せ替えられる。たとえば同じ封鎖術式でも、個人用では防御衣や携行結界器に入り、機械用では隔壁制御、検疫区画、港湾閉鎖系へ組み込まれる。

## まず結論

- `術式パッケージ` は機能設計単位、`配備束` は実装・認証単位である
- 同じ術式中核でも、`実行殻` と `接続子` を差し替えることで人間向けと機械向けに展開できる
- 人間向けパッケージは `判断補助` `感覚提示` `動作随伴` が重く、機械向けパッケージは `常駐安定性` `閉ループ制御` `安全側停止` が重い
- ISMOでは、汎用術式ライブラリの上に任務別パッケージが積まれ、それがさらに個人装備、艦載基盤、港湾設備、医療機器向けの配備束へ分かれる
- 高危険度分野では、術式本文よりむしろ `基準記相参照表` `禁止操作` `ロールバック点` `監査記録部` の方が重要になる
- これらの配備先が `装置` `機械` `施設` のどこに当たるかは、[魔導工学の実装階層](./魔導工学の実装階層.md) を参照

## 術式パッケージの構造

### 核術式束

- パッケージの中心となる機能群
- 基本操作プリミティブと高水準ユーティリティを、ある任務に向けてまとめたもの
- できる限り人間用と機械用で共通化される

### 基準記相参照表

- そのパッケージが通常参照する基準記相の一覧
- 乗員健常域、艦体標準姿勢、兵装安全域、検疫隔離基準などがここに入る

### 実行殻

- 核術式束を `誰がどう実行するか` を定める外殻
- 術者随伴型、端末補助型、機装常駐型、群制御型などの違いは主にここで決まる

### 接続子

- 術式を実際の身体、装備、機関、区画、センサー、アクチュエータへ束縛する接続規格
- 同じ核術式束でも、接続子が違えば手甲装備にも艦載砲にも載る

### 触媒調律表

- 必要な疑似解釈触媒の種別、調律値、交換周期、劣化時挙動を定める
- 人間向けでは短応答、機械向けでは長時間持続が重くなりやすい

### 安全制約部

- 出力上限、禁止対象、禁止操作、認証条件、安全側停止条件をまとめる
- 医療や検疫では最重視され、兵装でも無効化は重罪である

### 監査記録部

- 何を観測し、どの基準記相を引き、何を適用し、どこで止めたかを記録する
- 事故解析と責任追跡はここに依存する

### 起動条件部

- どういう状況で起動し、どういう条件で停止、待機、強制遮断するかを定める
- 常駐型パッケージでは特に重要になる

## 配備束

- 術式パッケージを、特定の対象へ認証済みの形で実装したもの
- 一つのパッケージから、複数の配備束が派生する
- たとえば同じ `防護封鎖パッケージ` でも、次のように分かれる

- 個人装備配備束
  防御衣、携行結界器、突入盾向け
- 艦載配備束
  区画隔壁、気密封鎖、損傷制御向け
- 港湾配備束
  接岸封鎖、検疫閉鎖、立入遮断向け

## 実行プロファイル

### 術者随伴型

- 解釈主体の中心を人間側が担う
- 手動判断、反応速度、感覚提示、個人差補正が重い
- 近接装備、携行端末、現場工兵具で多い

### 端末補助型

- 術者が発動意図を与え、実際の細部制御は端末側が肩代わりする
- 個人装備の大半はこの型に属する

### 機装常駐型

- 魔導機械や艦載基盤に常駐し、長時間の自律運転を行う
- 疑似解釈触媒、局所キャッシュされた署名済み基準記相断片、安全側停止が重要

### 環境常駐型

- 区画、港湾、工廠、病棟のような空間そのものに張り付く
- 人間が逐一触らず、空間条件の変化へ反応して動く

### 群制御型

- 複数の装置、艦、端末が同一パッケージを協調実行する
- 通信遅延、権限衝突、基準記相の同期ずれに弱いが、広域運用では不可欠

## 設計原則

### 機能と実装を分ける

- まず核術式束を決め、その後に実行殻と接続子を分ける
- これにより、一つの任務ロジックを人間と機械の双方へ再利用できる

### 人間向けは余裕を残す

- 術者随伴型では、個人適応ライブラリや現場判断が介入する余地を残す
- 代わりに、禁止操作と監査記録は強く維持する

### 機械向けは狭く深くする

- 機装常駐型では、扱う基準記相参照と操作範囲を狭く固定する
- 曖昧な高自由度より、長時間安定と安全停止が優先される

### 群制御では共通基準を絞る

- 群制御型では、全ノードが参照する基準記相と許容相を厳密に絞る
- ここが広すぎると、同期ずれがそのまま事故になる

## ISMO標準パッケージ群

### 1. 観測診断パッケージ

- 役割:
  対象の観測、束縛、照合、異常抽出、追跡印付け
- 核術式束:
  `観測` `束縛` `照合` `異常強調` `追跡維持`
- 主な基準記相参照:
  乗員健常域、艦体正常域、兵装整備域、汚染既知パターン
- 人間向け配備束:
  偵察端末、医療スキャナ、工兵診断具、照準補助眼鏡
- 機械向け配備束:
  艦内センサー網、機関診断台、検疫ゲート、港湾走査塔

### 2. 防護封鎖パッケージ

- 役割:
  結界、防御、区画閉鎖、汚染遮断、侵入抑止
- 核術式束:
  `同期` `固定` `隔離` `封鎖` `応急安定化`
- 主な基準記相参照:
  区画気密基準、防御衣安全域、封鎖境界、検疫閉鎖条件
- 人間向け配備束:
  個人結界器、防御衣、突入盾、携行封鎖杭
- 機械向け配備束:
  隔壁制御系、検疫区画、港湾閉鎖網、艦体損傷制御

### 3. 姿勢運動パッケージ

- 役割:
  姿勢補正、運動補助、反動吸収、接岸整列、照準安定
- 核術式束:
  `姿勢補正` `同期` `固定` `慣性差補正`
- 主な基準記相参照:
  人体運動許容域、艦体姿勢標準、砲座安定域、接岸姿勢基準
- 人間向け配備束:
  運動補助装備、反動制御銃、作業用外装具、降下支援具
- 機械向け配備束:
  艦体姿勢制御、砲塔安定化、荷役アーム、ドック誘導機

### 4. 通信航法パッケージ

- 役割:
  記相差分伝送、中継安定化、位置補正、航路整列、遭難捕捉
- 核術式束:
  `観測` `照合` `同期` `通信整列` `航路補正`
- 主な基準記相参照:
  航路標準、通信継路正常域、接岸基準、遭難識別規格
- 人間向け配備束:
  分隊中継器、遭難信号器、携行航法補助具
- 機械向け配備束:
  艦載通信核、航法演算基盤、標識浮標、中継局

### 5. 制圧兵装パッケージ

- 役割:
  目標束縛、命中補正、制圧、無力化、破壊
- 核術式束:
  `束縛` `命中補正` `切断` `攪乱` `固定` `破砕`
- 主な基準記相参照:
  兵装安全域、目標識別規則、交戦距離基準、誤射防止条件
- 人間向け配備束:
  銃器補助、近接制圧具、突入破城具、対装甲刺突器
- 機械向け配備束:
  艦載砲制御、近接防御火器、封鎖砲台、制圧ドローン
- 具体例は [制圧兵装パッケージ実例](./制圧兵装パッケージ実例.md) を参照

### 6. 救難医療パッケージ

- 役割:
  負傷安定化、浄化、生命維持、搬送補助、医療処置支援
- 核術式束:
  `安定化` `浄化` `同期` `隔離` `生命維持補助`
- 主な基準記相参照:
  生体健常域、外傷許容域、医療器具正常域、感染隔離基準
- 人間向け配備束:
  救命手甲、衛生兵端末、搬送補助具、野戦処置器
- 機械向け配備束:
  医療ポッド、病棟監視系、自動蘇生器、救難ドローン

### 7. 工兵整備パッケージ

- 役割:
  応急修復、荷役補助、配線再接続、仮設固定、回路迂回
- 核術式束:
  `観測` `束縛` `同期` `固定` `仮設接続`
- 主な基準記相参照:
  艦体構造標準、港湾接続規格、工業部材許容域、電力路正常図
- 人間向け配備束:
  工兵具、修繕端末、サルベージ補助具
- 機械向け配備束:
  工廠整備台、荷役制御系、保守フレーム、艦内自動修復系

### 8. 汚染対処パッケージ

- 役割:
  汚染検知、局所隔離、接続切断、損傷区画の秩序維持、封じ込め
- 核術式束:
  `観測` `照合` `隔離` `切断` `封鎖` `強制安定化`
- 主な基準記相参照:
  既知汚染相、侵蝕閾値、区画捨離条件、対エリミネーター閉鎖基準
- 人間向け配備束:
  対汚染防護具、封鎖杭、隔離マーカー、侵蝕対処端末
- 機械向け配備束:
  検疫遮断系、区画自壊防止系、汚染焼却設備、閉域防衛網

## 人間向けと機械向けの違い

### 人間向け

- 起動条件が比較的緩く、現場判断を受け入れる
- 感覚提示、照準補助、動作随伴の比重が高い
- 個人適応ライブラリによる差分が乗りやすい
- 一方で、術者負荷と誤操作の問題を抱える

### 機械向け

- 起動条件と停止条件が厳密で、許可された範囲しか触れない
- 高頻度再観測、閉ループ制御、長時間持続を前提にする
- 署名済み基準記相断片と疑似解釈触媒への依存が強い
- 一方で、未知現象への逸脱対応は人間やアリナ直結より弱い

## よくある複合パッケージ

- `突入複合束`
  観測診断 + 防護封鎖 + 制圧兵装
- `救難複合束`
  観測診断 + 救難医療 + 防護封鎖
- `艦載即応複合束`
  姿勢運動 + 制圧兵装 + 汚染対処
- `港湾検疫複合束`
  観測診断 + 防護封鎖 + 汚染対処 + 通信航法

## 制約

- すべてを一個の巨大パッケージへまとめると、査閲性も安全性も落ちる
- 人間向けの自由度をそのまま機械向けへ持ち込むと、常駐系事故を起こしやすい
- 機械向けの厳格さをそのまま人間向けへ持ち込むと、現場判断が死ぬ
- 結局、`核術式束は共通化し、実行殻は分ける` のが最も安定する
